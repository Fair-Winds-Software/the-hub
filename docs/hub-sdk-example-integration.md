# HUB SDK — Example integration (Manifest)

> **Status:** Placeholder walkthrough — the Fair Winds Manifest app is the intended reference integration but the cross-repo wiring lives in `~/development/manifest/` and is tracked by a separate Manifest-side ticket to be created at build time (see [S7 authoring note](https://fairwindssoftware.atlassian.net/browse/HUB-1824)). This document captures the intended shape so it lands the moment the Manifest side ships.

## Why Manifest?

Manifest is the simplest end-to-end example — it's an operational internal app with steady DAU and a small feature surface. Wiring it to HUB proves the full path (register → issue credentials → push metrics → see them in the HUB Console) without production billing risk.

## Registration

Done via the HUB Console (`/console/onboarding` → Register new), not manually via SQL:

| Field | Value |
|---|---|
| Tenant | Maverick Launch (internal umbrella) |
| Product name | Manifest |
| Slug | `manifest` |
| Product type | `internal_only` |

The wizard returns `(product_id, client_id, client_secret)` — the operator copies them into Manifest's `.env.local` and copies the generated Claude Code prompt into a Claude Code session inside `~/development/manifest/`.

## SDK bootstrap in Manifest

Expected path: `~/development/manifest/backend/src/hub/client.ts` (or platform-native equivalent).

Follows the pattern in `packages/hub-sdk/README.md` §3 verbatim — the Claude Code prompt generated by the wizard produces this file end-to-end.

## Metrics Manifest pushes

Given `product_type=internal_only`, Manifest exposes only these catalog names (`mrr_cents` + `churn_rate` filtered out per HUB-1821):

- `daily_active_users` — pushed once per day from Manifest's DAU-computation cron
- `logins` — pushed on each Manifest login-success
- `app_health_status` — pushed on Manifest's own uptime check tick

Where in Manifest's code:

| Metric | Site of push | Expected cadence |
|---|---|---|
| `daily_active_users` | end-of-day cron | 1×/day |
| `logins` | login-success handler | ~10-100/day |
| `app_health_status` | health check tick (every 5 min) | ~288/day |

## Verification

After first cron tick + one login:

1. In the HUB Console, open `/console/products/<manifest-product-id>/bi`.
2. Metric picker → `daily_active_users`.
3. Series should show 1+ point within ≤15 min of Manifest starting to push.

If nothing appears within the SLO window, the runbook (§Troubleshooting in the SDK README) is the first stop.
